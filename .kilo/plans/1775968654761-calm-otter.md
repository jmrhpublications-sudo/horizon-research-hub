# Plan: Update Submit Form for Normal Users to Upload PDF/DOC to Supabase Storage

## Overview
The goal is to modify the paper submission form for normal (non-logged-in) users to upload PDF or DOC files directly to Supabase storage, instead of requiring manual email attachment. These uploaded files should be visible in both the admin dashboard (with download capability) and the professor dashboard (with download capability).

## Current State Analysis
- **Submit Form (SubmitPaperPage.tsx)**: For non-logged-in users, the form collects data and prepares an email template, but does not upload files to storage. Logged-in users upload to Supabase 'papers' bucket.
- **Admin Dashboard (AdminPapers.tsx)**: Displays submissions with attachments. Supports ZIP download and individual file downloads for both Supabase and Google Drive files.
- **Professor Dashboard (ProfessorDashboard.tsx)**: Shows assigned papers with "Download PDF" functionality using signed URLs.
- **Database**: `papers` table has `attachments TEXT[]` field for storing file URLs/paths.
- **Storage**: Uses Supabase storage bucket 'papers'. Google Drive upload function exists but appears used for professor submissions.

## Proposed Changes

### 1. Update Submit Form (SubmitPaperPage.tsx)
- Modify the file upload logic to upload files to Supabase storage for non-logged-in users.
- Generate unique submission IDs and store files under `papers/{submission_id}/` path.
- Update the `submitPaper` function to insert submission data into `papers` table with `attachments` array containing uploaded file URLs.
- Maintain email notification to admin with submission details (but files are now uploaded, not attached to email).

### 2. Verify Admin Dashboard Compatibility
- Confirm AdminPapers.tsx correctly displays and allows downloading of Supabase-stored attachments.
- Test ZIP download and individual file download functionality.

### 3. Update Professor Dashboard (ProfessorDashboard.tsx)
- Ensure the dashboard can display all attachments for assigned papers.
- If multiple files are uploaded, provide download options for each attachment.
- Update the "Download PDF" button to handle multiple files or provide a comprehensive download option.

### 4. Database and Storage Updates
- Ensure `attachments` field in `papers` table stores full Supabase file paths/URLs.
- Update any related functions (e.g., `notify-admin-submission`) if needed for the new upload process.

### 5. Testing and Validation
- Test end-to-end submission process for non-logged-in users.
- Verify file uploads to Supabase storage.
- Test admin and professor dashboard display and download functionality.
- Validate file type restrictions (PDF, DOC, DOCX) and size limits (100KB-20MB).

## Questions for Clarification
1. Should the form allow multiple file uploads (PDF/DOC), or only one main file?
2. For professor dashboard, if multiple files are attached, should it show all downloadable files or just the primary PDF?
3. Are there any specific naming conventions or folder structures needed for the uploaded files?
4. Should the email notification include links to the uploaded files instead of attachments?

## Implementation Steps
1. Modify SubmitPaperPage.tsx to handle file uploads for non-logged-in users.
2. Update submission logic to insert into database with attachments.
3. Test admin dashboard download functionality.
4. Update professor dashboard to display and allow downloads of attachments.
5. Full end-to-end testing.

## Risks
- Potential storage costs increase with file uploads.
- Need to ensure proper file validation and security.
- Compatibility with existing Google Drive integrations if any overlap.
